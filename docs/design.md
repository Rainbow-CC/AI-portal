# AI 中台 (AI-Portal) 项目设计文档

## 1. 项目概述
AI 中台旨在构建一个统一的 AI 能力集成与管理平台，通过 AI 网关屏蔽总行 API 细节，为公司内部系统提供标准、稳定的 AI 能力接口。同时提供可视化前台，辅助开发者了解能力分布、监控调用情况及参考应用案例。

## 2. 系统架构

### 2.1 技术栈
- **后端框架**: Java 17+, Spring Boot 3.x, Spring Cloud 
- **前端框架**: React 18, Ant Design / Tailwind CSS
- **通信协议**: HTTP/1.1 & HTTP/2 (支持 SSE 流式返回)
- **数据交互**: 内部标准 JSON 协议（自研）
- **指标监控**: Spring Boot Actuator + Prometheus (可选)

### 2.2 逻辑架构
1. **AI 网关层**: 基于 Spring Cloud Gateway 实现，负责请求分发、协议转换、权限校验。
2. **前台展现层**: 提供应用案例展示、监控看板、模型能力地图。
3. **外部 API 适配**: 对接总行 Public API，进行请求透传与异常处理。

## 3. 功能需求

### 3.1 AI 网关
- **统一对接**: 屏蔽总行 API 细节，内部业务方仅需对接一套自研 JSON 协议。
- **SSE 支持**: 网关层必须支持 Server-Sent Events (SSE) 的透明转发，以满足 LLM 逐字生成的交互需求。
- **请求转发**: 接收内部系统请求，添加必要参数，转发至 Public API。
- **鉴权管理**: 采用 **API Key** 方式。请求头需携带 `X-API-Key`，网关进行合法性校验，不涉及复杂的多租户隔离。

### 3.2 前台页面
- **监控大盘**:
    - 实时展示系统 **TPS** (每秒请求数)。
    - 统计网关响应 **延迟 (Latency)**。
    - 展示服务 **可用率 (Availability)**。
- **能力地图**:
    - **展现形式**: 卡片式布局，每个卡片代表一个模型能力。
    - **内容详情**: 点击卡片弹出 Modal，展示接口介绍、API 定义文档。
    - **在线调试**: 提供跳转链接，指向已有的在线 Playground 环境。
- **应用案例分享**: 展示内部已落地的 AI 应用场景，提供图片、文字描述。

## 4. 非功能需求

### 4.1 性能与可扩展性
- **吞吐量**: 系统需支持 **100 TPS** (总和)，并具备水平扩展能力以应对未来流量增长。
- **延迟**: 网关自身的处理损耗（从接收请求到发出请求的 OverHead）必须控制在 **100ms 以内**（不含大模型逻辑推理时间）。
- **可靠性**: 需具备高可用性，确保在单点故障时能够快速切换，支持多副本部署。

### 4.2 运维约束
- **部署要求**: 不强制要求部署在 Kubernetes 环境下，支持常规虚机/容器部署。
- **容灾策略**: 暂不考虑跨机房容灾，重点保障单集群内的高可用性。

## 5. 限制条件 (Constraints)
- **不考虑**: 安全审计、内容脱敏、Token 消耗统计、费用分摊、Prompt 缓存、后端自动切换（模型路由）。

## 6. API 设计原则 (自研协议示例)
- **请求**: `POST /v1/ai/chat` (Header: `X-API-Key: xxx`)
- **响应**: 统一返回结构体，包含 `code`, `message`, `data`。
- **流式**: `Content-Type: text/event-stream`。
